iPadだけでGitHub Pagesに個人サイトを公開する

iPadだけを使い、GitHub Pages上に個人サイトを公開しました。 最初の目的は、複雑な機能を実装することではなく、 後から継続的に更新できる最小構成を作ることでした。

使用したもの

  • iPad
  • Safari
  • GitHubアカウント
  • GitHub Pages
  • github.dev
  • HTML
  • CSS

最初に作成した構造

サイトは、トップページから複数のページへ移動できる構造にしました。 

sirkerf.github.io/
├── index.html
├── profile.html
├── works.html
├── writing.html
├── style.css
└── articles/
    └── ipad-github-pages.html

公開までの手順

  1. GitHub上に sirkerf.github.io リポジトリを作成する。
  2. GitHub Pagesの公開元として main ブランチの /(root) を指定する。
  3. index.html を作成する。
  4. style.css を追加する。
  5. Profile、Works、Writingの各ページを追加する。
  6. 変更をコミットしてGitHubへ反映する。

iPadで編集する際の考え方

iPadでは、ブラウザ上で動く github.dev を使ってファイルを編集しました。 複数のHTMLファイルとCSSファイルを扱う場合、 通常のGitHub画面よりも編集しやすくなります。

また、変更内容は一定の単位でコミットします。 一度に大量の変更を加えるよりも、 ページ単位で履歴を残した方が修正しやすくなります。
